### Mode 1: VPIN風控 🤖🟡

**設計目的**: 只使用市場毒性指標進行風控

**配置**:
```json

    "vpin_threshold": 0.85,
    "liquidity_check": false
  }
}
```

**特點**:
- 🟡 只檢查 VPIN (Volume-synchronized Probability of Informed Trading)
- 🚫 VPIN > 0.85 時拒絕交易
- 📊 單一指標風控
- 實測: -1.68% (5筆交易, 20%勝率)

## ⚠️ 風險提示

### 所有策略的共同限制

1. **API 延遲**: 101ms 平均延遲，不適合真正 HFT
2. **手續費**: Taker 0.05%，頻繁交易會侵蝕利潤
3. **滑點**: 高頻交易滑點約 2-5 bps
4. **市場變化**: 歷史表現不代表未來結果

### Mode 13 特別注意

- ⏳ **未在低毒性市場驗證**: 目前只在 VPIN > 0.85 環境測試
- ⏳ **長期持倉風險**: 8 小時持倉可能遭遇市場突變
- ⏳ **多時間框架同步**: 需要較長數據累積期（2-4天）

### 技術指標策略 (M8, M9)

- ⚠️ **逆勢邏輯**: 技術指標多為逆勢，可能與 OBI 順勢衝突
- ⚠️ **滯後性**: 基於歷史價格，領先性不如 OBI
- ⚠️ **過度過濾**: M9 可能整天無交易
